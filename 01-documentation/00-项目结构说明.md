## 📋 文件分类说明

### 🎯 核心文件（根目录）

**作用**：应用运行必需的文件

- `index.html` - 主页面，应用入口
- `style.css` - 全局样式表
- `js/` - 所有 JavaScript 代码模块

**特点**：
- ✅ 必须保留在根目录
- ✅ 不要移动位置
- ✅ 这些是应用的核心

---

### 📚 01-documentation/ - 项目文档

**作用**：所有项目相关的说明文档

**编号规则**：
- `00-` 开头：项目结构、索引类文档
- `01-` 开头：主要文档（README、架构等）
- `05-` 开始：功能说明文档
- `09-` 开始：操作指南文档

**主要文档**：
- `01-README.md` - 项目概述和使用指南
- `02-ARCHITECTURE.md` - 技术架构详解
- `03-GETTING_STARTED.md` - 快速上手指南
- `05-数据存储说明.md` - 数据存储位置和操作
- `06-如何更新Demo数据.md` - Demo 数据更新方法
- `10-公司电脑-下载和启动指南.md` - 公司电脑操作指南

**访问方式**：
- 直接打开 `.md` 文件查看
- 或在 GitHub 上查看

---

### 🛠️ 02-tools/ - 工具文件

**作用**：辅助工具和脚本

**文件说明**：
- `data-manager.html` - 数据管理工具
  - 访问：`http://localhost:8000/02-tools/data-manager.html`
  - 功能：查看、清空、导出/导入数据

- `start.sh` - 启动脚本
  - 用法：`./02-tools/start.sh`
  - 功能：快速启动本地服务器

- `update.sh` - 更新脚本
  - 用法：`./02-tools/update.sh`
  - 功能：快速更新代码到 GitHub

---

### 💾 03-backup/ - 备份文件

**作用**：保存旧版本或备份文件

**文件说明**：
- `app.js.backup` - 重构前的原 app.js 备份
  - 仅作参考，不参与运行

---

### 📁 docs/ - 业务文档

**作用**：你的业务相关文档（不参与代码）

**特点**：
- ❌ 不上传到 GitHub（已在 .gitignore 中排除）
- ✅ 仅本地保留
- ✅ 与代码无关

## 📝 文件命名规则

### 文档文件
- 格式：`编号-文件名.md`
- 编号：00-99，按重要性排序
- 中文文件名：使用中文，便于理解

### 工具文件
- 脚本：`.sh` 扩展名
- HTML 工具：`.html` 扩展名
- 描述性命名

### 备份文件
- 格式：`原文件名.backup`
- 或：`原文件名.old`

---

## 🔄 如何添加新文件

### 添加新文档
1. 放入 `01-documentation/`
2. 使用编号：`12-新文档名.md`
3. 更新本文档的目录

### 添加新工具
1. 放入 `02-tools/`
2. 使用描述性名称
3. 更新本文档说明

### 添加新备份
1. 放入 `03-backup/`
2. 使用 `.backup` 或 `.old` 扩展名
